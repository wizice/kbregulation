데이터베이스 설계 방안

주요 테이블 구조

1.1.1 관리자 테이블 (ADMIN_USER)

필드명 데이터 타입 제약조건 설명
ADMIN_ID VARCHAR(20) PK, NOT NULL 관리자 고유 ID
USER_ID VARCHAR(50) UNIQUE, NOT NULL 사용자 ID
PASSWORD VARCHAR(50) NOT NULL 비밀번호
USER_NAME VARCHAR(100) NOT NULL 사용자명
EMAIL VARCHAR(200) NOT NULL 이메일
DEPARTMENT VARCHAR(100) 소속 부서
ROLE VARCHAR(20) NOT NULL 역할 (시스템관리자/내규관리자/부서관리자)
AUTH_LEVEL INT DEFAULT 1 권한 레벨 (1-9)
CREATED_DATE DATETIME DEFAULT NOW() 생성일
LAST_LOGIN DATETIME 최종 로그인
STATUS CHAR(1) DEFAULT 'Y' 상태 (Y:활성/N:비활성)

1.1.2 내규 마스터 테이블 (REGULATION_MASTER)

필드명 데이터 타입 제약조건 설명
REG_ID VARCHAR(20) PK, NOT NULL 내규 고유 ID
REG_CODE VARCHAR(20) UNIQUE, NOT NULL 내규 코드 (예: 1.1.1)
REG_TITLE VARCHAR(500) NOT NULL 내규 제목
CATEGORY_LEVEL1 VARCHAR(10) NOT NULL 대분류 코드
CATEGORY_LEVEL2 VARCHAR(10) 중분류 코드
CATEGORY_LEVEL3 VARCHAR(10) 소분류 코드
CURRENT_NOTICE_NO VARCHAR(50) 현재 공포 번호
CURRENT_VERSION_ID VARCHAR(20) FK 현재 유효한 버전 ID
STATUS VARCHAR(10) DEFAULT '유효' 상태 (유효/폐기/개정중)
UPLOAD_ADMIN_ID VARCHAR(20) FK, NOT NULL 업로드한 관리자 ID
CREATED_DATE DATETIME DEFAULT NOW() 생성일
LAST_UPDATED DATETIME 최종 수정일

1.1.3 내규 버전 테이블 (REGULATION_VERSION)

필드명 데이터 타입 제약조건 설명
VERSION_ID VARCHAR(20) PK, NOT NULL 버전 고유 ID
REG_ID VARCHAR(20) FK, NOT NULL 내규 ID
NOTICE_NO VARCHAR(50) 공포 번호
REVISION_DATE DATE 개정일
EFFECTIVE_DATE DATE 시행일자
APPENDIX_IDS TEXT 해당 버전의 부록 ID 목록 (쉼표 구분)
FILE_PATH VARCHAR(500) NOT NULL 파일 경로
FILE_NAME VARCHAR(300) NOT NULL 원본 파일명
PARSED_CONTENT LONGTEXT 파싱된 내용 (검색용)
REVISION_REASON TEXT 개정 사유
UPLOAD_ADMIN_ID VARCHAR(20) FK, NOT NULL 업로드한 관리자 ID
AUTO_PARSED CHAR(1) DEFAULT 'N' 자동 파싱 여부 (Y/N)
PARSING_STATUS VARCHAR(10) DEFAULT '대기' 파싱 상태 (성공/실패/진행중/대기)

1.1.4 부록 테이블 (APPENDIX)

필드명 데이터 타입 제약조건 설명
APPENDIX_ID VARCHAR(20) PK, NOT NULL 부록 고유 ID
REG_ID VARCHAR(20) FK, NOT NULL 연결된 내규 ID
APPENDIX_CODE VARCHAR(20) NOT NULL 부록 코드 (예: 부록1, 부록2)
APPENDIX_TITLE VARCHAR(500) NOT NULL 부록 제목
FILE_PATH VARCHAR(500) NOT NULL 파일 경로
FILE_NAME VARCHAR(300) NOT NULL 원본 파일명
PARSED_CONTENT LONGTEXT 파싱된 내용 (검색용)
UPLOAD_ADMIN_ID VARCHAR(20) FK, NOT NULL 업로드한 관리자 ID
AUTO_PARSED CHAR(1) DEFAULT 'N' 자동 파싱 여부 (Y/N)
PARSING_STATUS VARCHAR(10) DEFAULT '대기' 파싱 상태 (성공/실패/진행중/대기)
CREATED_DATE DATETIME DEFAULT NOW() 생성일
UPDATED_DATE DATETIME 수정일

1.1.5 분류 체계 테이블 (CATEGORY)

필드명 데이터 타입 제약조건 설명
CATEGORY_ID VARCHAR(10) PK, NOT NULL 분류 ID
PARENT_ID VARCHAR(10) FK 상위 분류 ID
CATEGORY_CODE VARCHAR(10) NOT NULL 분류 코드
CATEGORY_NAME VARCHAR(200) NOT NULL 분류명
LEVEL INT NOT NULL 분류 레벨 (1,2,3)
SORT_ORDER INT DEFAULT 0 정렬 순서

1.1.6 문서 업로드 로그 테이블 (UPLOAD_LOG)

필드명 데이터 타입 제약조건 설명
LOG_ID VARCHAR(20) PK, NOT NULL 로그 고유 ID
ADMIN_ID VARCHAR(20) FK, NOT NULL 업로드한 관리자 ID
FILE_NAME VARCHAR(300) NOT NULL 업로드된 파일명
UPLOAD_DATE DATETIME DEFAULT NOW() 업로드 일시
PARSING_RESULT VARCHAR(10) 파싱 결과 (성공/실패)
ERROR_MESSAGE TEXT 오류 메시지 (있을 경우)
REG_ID VARCHAR(20) FK 생성된 내규 ID (성공 시)
APPENDIX_COUNT INT DEFAULT 0 연결된 부록 수
